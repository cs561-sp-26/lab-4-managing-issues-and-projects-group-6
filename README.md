## Jira vs. GitHub Projects
### Pros of Jira
- Jira has strong support for agile workflows, including sprints, backlogs, story points, and issue tracking. It is built around project management, so it gives teams a lot of structure.
- Jira supports a detailed issue hierarchy with epics, stories, tasks, and subtasks. This makes it easier to organize larger projects with many related pieces of work.
- Jira has more advanced workflow and reporting tools. Teams can customize status, track progress, and generate reports more easily.

### Cons of Jira
- Jira has more setup involved and can take longer to learn. For a smaller lab project, it may be more tool than we actually need.
- Jira is not as directly connected to the GitHub repository. Linking commits, branches, pull requests, and issues takes more setup.
- Jira can feel heavier for smaller teams. A lot of its features are useful, but they can also slow things down when the project is simple.

### Pros of GitHub Projects
- GitHub Projects connects directly to the repository, issues, branches, commits, and pull requests. This makes it easier to keep the code and project tracking in the same place.
- GitHub Projects is simpler to set up and use. The Kanban board, issue assignments, and roadmap view are straightforward enough for this lab.
- GitHub makes it easy to link issues to branches and pull requests. This helps show a clear path from user story to implementation.

### Cons of GitHub Projects
- GitHub Projects has fewer built-in agile features than Jira. Some things, like story points and sprint tracking, have to be added manually.
- GitHub Projects has less advanced reporting. It works well for tracking issues, but it is not as strong for detailed project metrics.
- GitHub Projects has fewer workflow customization options. It is good for a simple sprint, but it may be limiting for a larger software project.

## Tool Chosen
Github Projects

## Rationale
GitHub Projects was chosen because it integrates directly with our repository, allowing us to easily link issues, commits, branches, and pull requests. This made it easier to track progress and maintain a clean workflow without needing to switch between tools. Since our project is relatively small, GitHub Projects provided all the necessary functionality with less overhead than Jira, while still supporting Kanban tracking, issue assignment, and collaboration. The 1 week time limit was also another major factor in choosing GitHub Projects.
